Hexo 和 Github 搭建博客

因为在学习数据结构与算法的过程中，老师要求刷 LeetCode 题目并撰写博客，于是开始了 CSDN 博客之旅。但是个人并不喜欢 CSDN 的博客，而自己搭建一个网站服务能力有限，便想搭建一个静态网站，Github 和 Hexo 便是我的首次选择。这篇博客记录了搭建博客的过程。

Hexo

什么是 Hexo？

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown (或其他渲染引擎) 解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

安装前提

访问 Node.js 的官网下载需要的版本(版本不低于 8.10，建议使用 10.0 及以上版本)，

访问 Git 的官网下载需要的版本。

检验软件是否安装成功：

安装 Hexo

所有必备的应用程序安装完成后，即可使用 npm 安装 Hexo。

npm install -g hexo-cli

安装 hexo 部署到 git page 的 deployer。

npm install hexo-deployer-git --save

Hexo 初始化

首先，进行初始化

hexo init

结果发现执行错误：

查阅相关信息，发现是 powershell 的执行策略处于 Restricted 状态，以管理员身份运行 powershell 更改 powershell 的执行策略如下：

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned

然后，继续初始化。安装完成后，将会在指定文件夹中新建所需要的文件。目录如下：

查看效果

执行以下命令，执行完后即可登录 http://localhost:4000/ 查看效果

hexo generate
hexo server

Github Pages

现在，已经完成了本地博客的搭建，我们只能通过本地连接才能查看博客。现在，我们要将博客部署到 Github Pages 上，实现网络上的访问。

首先，登录上我的 Github 账户。以 用户名.github.io 为项目名创建项目代码库。

在测试连接 GitHub 的过程中，出现了错误：

查阅网上资料后，在存放公钥私钥(id_rsa 和 id_rsa.pub)的文件里，新建 config 文本，内容如下：

Host github.com
User YourEmail@163.com
Hostname ssh.github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa
Port 443

再次测试，成功。

将本地 Hexo 文件更新到 GitHub 库中

登录 GitHub 打开新建的项目，复制 SSH 地址，然后修改 _config.yml 文件：

  • 找到 deploy 关键字，type 后面写上 git；
  • 补充 repository 并粘贴 SSH 地址；
  • 补充 branch，如果没有改动分支就填写 master(GitHub 的 master 好像改成了main)。

注意：所有冒号之后都有一个空格。

# Deployment
## Docs: https://hexo.io/dics/deployment.html
deploy:
  type: git
  repository: git@github.com:yourname/yourname.github.io.git
  brance: main

在 Hexo 文件夹中执行以下命令：

hexo g
hexo d

或者直接执行

hexo g -d

执行完后，发现没有找到 Deployer

重新执行以下命令：

nmp install hexo-deployer-git --save

成功后，执行 hexo g -d 完成，这样博客就会部署到 GitHub 上了。

最后，就可以通过地址 https://用户名.github.io 来访问博客了。

如何在博客上发表文章

现在，我们已经搭建好了属于自己的博客。下面就是如何在博客上发表文章的问题。

首先，输入以下命令新建一个空文章：

hexo n "文章标题"

会在项目 \Hexo\source\_posts 中生成 文章标题.md 文件。

也可以直接在 \Hexo\source\_posts 目录下新建 Markdown 文档。

当我们编辑好了文章后，使用以下命令将其推送到服务器上：

hexo d -g

现在访问你的博客就可以看到写好的文章了。
